Tokenizer Ekspresi Aritmatika DFA - maximal munch

Satu DFA tunggal membaca ekspresi karakter demi karakter. Setiap state akhir ditandai kategori token (NUMBER, PLUS, MINUS, STAR, SLASH, LPAREN, RPAREN). Strategi maximal munch dipakai persis seperti lexer pada compiler sungguhan: ambil kecocokan terpanjang sebelum berhenti.

Lingkaran bergaris tebal berwarna kuning menandakan state akhir (token terbentuk di situ). Jalankan tokenisasi dulu, lalu pakai tombol putar di bawah untuk melihat state mana yang dilewati untuk tiap token, satu per satu.

Konversi NFA → DFA Subset Construction

Mengubah NFA (boleh memuat transisi-ε) menjadi DFA ekuivalen. Setiap state DFA merepresentasikan himpunan bagian (subset) dari state-state NFA.

Preset: NFA mengandung "aa" atau "bb" Preset: NFA pola bilangan (ε-transisi)
Tiap baris berarti: dari himpunan state DFA di kolom pertama, kalau baca simbol di kolom kedua, gabungan tujuan semua state NFA di dalamnya menghasilkan himpunan baru di kolom ketiga. Himpunan-himpunan inilah yang jadi state pada DFA hasil konversi.

Kompilasi Regular Expression Thompson NFA → Subset Construction DFA

Mendukung: literal, | (union), * + ?, grup (), character class [a-z0-9], shorthand \d \w \s. Contoh pola untuk token NUMBER pada tokenizer modul 1: [0-9]+(\.[0-9]+)?

[0-9]+(\.[0-9]+)? (a|b)*abb \d\d?:\d\d
NFA di kiri adalah hasil langsung dari Thompson Construction, satu simpul untuk tiap operator dalam pola (makanya jumlah state-nya cenderung banyak dan transisi-nya banyak memakai ε). DFA di kanan adalah bentuk yang sudah disederhanakan lewat subset construction, dipakai untuk pengujian string di bagian bawah.

Simulasi Pushdown Automaton (Shift-Reduce)
SHIFT berarti mendorong satu token dari sisa input ke atas stack. REDUCE berarti beberapa simbol teratas stack cocok dengan ruas kanan sebuah produksi, lalu diganti (di-"reduce") jadi ruas kirinya. Kalau proses berakhir dengan stack cuma berisi simbol awal E dan input sudah habis, ekspresi tersebut valid menurut grammar.
